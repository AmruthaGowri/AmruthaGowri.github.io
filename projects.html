Flagship Research

Multi-Trigger Backdoor Attacks in Code-Generating LLMs

This project studies how code-generating large language models can be poisoned using hidden triggers during LoRA fine-tuning. The work targets CodeGen-350M and examines how semantic comments and syntactic code transformations behave differently as backdoor triggers.

What I Did

  • Designed a multi-trigger backdoor poisoning framework.
  • Used LoRA adapters for parameter-efficient fine-tuning.
  • Generated poisoned datasets from CodeSearchNet.
  • Tested poisoning rates from 1% to 20%.
  • Built PyTorch and HuggingFace evaluation pipelines.
  • Conducted activation inspection and weight differential analysis.

Key Findings

  • Multiple triggers can coexist in one LoRA adapter.
  • Semantic triggers were highly specific and separable.
  • Syntactic triggers caused distributional confusion.
  • Standard code quality metrics may not reveal hidden backdoors.
  • LoRA-level surgical defense can remove malicious behavior without retraining the full base model.

Responsible AI

RAIMA — Responsible AI Monitoring and Auditing Assistant

RAIMA is a modular AI auditing system for evaluating fairness, explainability, hallucination risks, and deployment readiness in ML and LLM systems.

Features

  • SHAP and LIME explanations.
  • Fairness evaluation dashboards.
  • Demographic parity, equal opportunity, and disparate impact metrics.
  • Streamlit-based real-time monitoring interface.
  • LLM-assisted risk report generation.

Impact

RAIMA helps teams understand how AI systems make decisions and whether those systems are explainable, fair, and safe for deployment.
